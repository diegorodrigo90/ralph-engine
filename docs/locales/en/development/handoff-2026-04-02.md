# Handoff — 2026-04-02

## Current state

- The working tree is clean on `main`.
- Recent work focused on consolidating the Rust-first reboot around typed runtime contracts, typed locale contracts, scaffold/runtime alignment, and stronger public package documentation.
- CI, release promotion, SonarCloud, and public-surface publishing are in a much healthier state than earlier in the reboot.

## Recent completed slices

- Shared runtime hook and capability contracts were moved into `re-core`, reducing command-local runtime branching in `re-cli`.
- `re-config` now owns a typed shared locale contract (`SupportedLocale`) used by `re-cli`, `re-core`, `re-plugin`, and `re-mcp`.
- Public plugin locale getters now resolve locale through the typed shared contract instead of raw locale-string handling.
- The plugin scaffolder now owns a dedicated runtime-surface catalog in `tools/create-ralph-engine/lib/runtime-surfaces.js` instead of embedding a large capability-to-hook switch directly in the bin script.
- Public `README`s for the repository, npm packaging, Homebrew packaging, and `create-ralph-engine-plugin` were strengthened with badges, open-source guidance, and clearer release/distribution messaging.
- Shared-crate contract tests were decoupled further from official plugin identities, while official plugin crates now own closer manifest and contribution verification.

## Important current constraints

- npm and Homebrew publication are still intentionally gated. The README badges describe them as gated channels on purpose.
- The public npm package name `ralph-engine` still has legacy history outside the Rust reboot, so npm badges should stay explicit and honest until the Rust channel is the real published channel.
- CLI, plugin metadata, and scaffolded plugins currently support `en` and `pt-br`, with English fallback when a locale catalog is missing.
