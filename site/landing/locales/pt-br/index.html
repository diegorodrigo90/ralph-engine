Runtime open source orientado a plugins

O runtime para fluxos de desenvolvimento com agentes.

Ralph Engine entrega uma base em Rust para loops de agente, plugins oficiais, integrações com MCP e verificações de qualidade que continuam coerentes do desenvolvimento local até a CI.

Instalação via código-fonte

git clone https://github.com/diegorodrigo90/ralph-engine.git
cd ralph-engine
./scripts/bootstrap-dev.sh
cargo run -p re-cli -- --version

npm e Homebrew continuam canais oficiais e estão sendo reconstruídos no novo pipeline de release.

Por que Ralph Engine

Um runtime deve apertar o workflow, não adicionar mais uma camada frágil.

Ralph Engine está sendo construído para times que valorizam clareza, manutenção e disciplina operacional. O objetivo não é mais um wrapper raso sobre modelos. O objetivo é um runtime durável.

Arquitetura plugin-first

Plugins oficiais evoluem em Rust, plugins de terceiros continuam agnósticos de linguagem, e capabilities seguem como unidade de extensibilidade.

  • Capabilities em vez de pontos de extensão ad hoc
  • MCP externo continua como parte nativa da arquitetura
  • Templates continuam sob responsabilidade do plugin

Qualidade e segurança como features do produto

SemVer, Actions pinadas, varredura de supply chain, linting estrita, cobertura de testes e automação de release fazem parte do contrato do runtime.

  • Rustdoc, clippy, testes, coverage e docs build
  • cargo-audit, cargo-deny, Trivy e gitleaks
  • Artefatos assinados, SBOM e provenance no release

Contexto que continua coerente

Docs, catálogo, metadados do repositório e superfícies legíveis por máquina ficam alinhados de propósito para que pessoas e agentes encontrem o contexto certo rapidamente.
