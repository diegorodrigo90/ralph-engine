Why Ralph Engine

A runtime should tighten workflows, not add another fragile layer.

Ralph Engine is being built for teams that care about clarity, maintainability, and operational discipline. The point is not another thin wrapper around models. The point is a durable runtime.

Plugin-first architecture

Official plugins evolve in Rust, third-party plugins stay language-agnostic, and capabilities remain the unit of extensibility.

  • Capabilities instead of ad hoc extension points
  • External MCP remains first-class
  • Templates stay plugin-owned

Quality and security as product features

SemVer, pinned Actions, supply-chain scanning, strict linting, coverage targets, and release automation are part of the runtime contract.

  • Rustdoc, clippy, tests, coverage and docs build
  • cargo-audit, cargo-deny, Trivy and gitleaks
  • Signed artifacts, SBOM and provenance in the release path

Context that stays coherent

Docs, catalog, repo metadata and machine-readable surfaces are intentionally aligned so humans and agents can find the right context fast.
